관계 모델
	-각 데이터 모델들 간의 관계를 표현
	attribute : 스키마에서 각 항목들
	튜플: 하나의 데이터 셋  
소형화 되어가는 시스템
	DBMS 비쌋지만 , 오늘날에는 저렴하다.
Database 응용의 구조
	클라이언트 - 서버 (2티어) 구조를 사용했으나 보안적으로 문제
	
		멀티 티어 
			- 클라이언트,웹서버,모바일서버 

			- 3티어 시스템	(프레젠테이션 티어) - 로직 티어 - 데이터 티어

		
	2티어 서버 : 클라이언트와 DB 사이의 Path가 유출 되면 위험
	
	멀티티어 환경 : 클라이언트와 web서버의 중간서버들이 연결  그리고 이러한 중간서버가 클라이언트에 연결
			(따라서 클라이언트가 중간서버들을 거쳐서 DB에 접근 하기에 2티어보다 안전)
멀티미디어 데이터베이스
	- 텍스트만 저장이 아니라 이미지,오디오,비디오,문서
	- 대용량이며 크기도 다양함
	- 기가바이트의 객체나 튜플을 수용
	- 상호 대화식으로 응답의 일부를 전달하거나, 또는 미리 정의된 속도로 응답을 전달

데이터 웨어하우스
	- 한 기관내에 여러 데이터베이스들을 관리
	- 데이터 마이닝 데이터 웨어하우스에서 원하는 정보 뽑기

DBMS의 구성요소의 개요
	- 질의 : 데이터에 관한 질문
	- 데이터 변경 : 데이터를 변경하기 위한 연산
	- 스키마 변경 : 메타데이터(데이터구조)를 변경하기 위한 연산(잘일어나지 않는다)
	- 메타 데이터 : 데이터의 구조에 대한 정보
