운영 체제는 무엇인가?
	애플리케이션,하드웨어의 중간 다리

운영 체제의 목표
	유저 프로그램 쉽게 실행,효과적으로 관리

컴퓨터 시스템 4가지 구조
	
	1.하드웨어
	2.OS
	3.어플리케이션
	4.유저(사람,머신)
	user -> system and application -> OS -> hardware

운영체제가 하는일
	1) 쉬운 사용과 좋은포퍼먼스 관리
	2) 좋지않은 리소스를 최적화해서 사용가능하게
	3) os는 메모리 할당역할
	4) os는 프로그램 구동

OS를 통용하는 정의를 내리기 힘들다.
	하지만) 컴퓨터 속에서 항상 돌고있는 하나의 프로그램 커널(kernel os핵심 기능)
	           

컴퓨터 start-up
	전원-> firmware start -> kernel에 os 로딩 -> 프로세스 구동 준비

컴퓨터 시스템 동작
	1) I/o device 와 cpu는 동시 구동 가능
	2) 각 장치 컨트롤러가 특정 장치 유형을 담당함
	3) 각 장치 컨트롤러에 로컬 버퍼가 있음
	4) CPU가 데이터를 메인 메모리에서 로컬 버퍼로/로 이동
	5) I/O가 디바이스에서 컨트롤러의 로컬 버퍼로
	6) 디바이스 컨트롤러가 CPU의 작동이 중단됨을 알려줌
인터럽트
	급한일이나 다른일들이 있으면 멈추고 일처리(컴구조에서 배운 개념)
	
	소프트웨어적 인터럽트 = trap, exception(ex: 0으로 나누기 연산)
