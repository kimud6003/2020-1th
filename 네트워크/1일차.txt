네트워크 계층별로 어떤일을 하는가

네트워크를 왜하는것인가? = 통신(데이터 주고/받기)을 잘하기위해서

데이터를 잘 주고 받는다 = 데이터 무결성 기준 and 속도 and 많은양

네트워크 : 상호연결이 가능한 통신 장치의 집합체
	
	그물처럼 서로 연결되어 통신을 하는 형태


네트워크의 종류:	근거리통신망

		광역통신망

		네트워크간 네트워크

근거리 통신망 : 사무실,빌딩, 학교등에서 호스틀을 개별적으로 연결하여 사용

	        각 호스트를 나타내는 식별자 사용

	        각 호스트들과 자원을 공유

		과거 LAN 방식 : 하나의 손으로 Host들이 연결된 방식

		현대 LAN 방식 : switch에 연결해서 공유

		Point to Point :  전송매체를 통하여 두개의 통신장치를 연결

		switch WAN: 스위치가 있어서 서로 통신
		
		두 개의 LAN과 하나의 WAN으로 구성된 네트워크간 네트워크
		:각각의 네트워크를 Router로 통신을한다.

		여러개의 Wan과 Lan으로 구성된 혼합네트워크

교환은 그냥 읽어보거라!

	회선 교환망: 두종단 시스템 사이에 회선이라는 전용선 사용
			장점이 곧 단점이 된다.

	패킷 교환망: 두종단 사시의 통신은 패킷에 의해 이루어짐
			단점: 과부화 걸리면 끊기거나 망가짐

인터넷 
	백본 : 굉장히 큰 교환 시스템

인터넷에 접속 : 물리적으로 ISP에 연결

인터넷 접속 종류 4가지 : 이런게 있다 하고 넘어감

프로토콜 계층화: 송신자와 수신자가 따라야 하는 규칙
	          통신이 간단 할때는 하나의 프로토콜이 필요

단일 계층 프로토콜: 우리가 일상에서 사용 말하는 것과 유사

3- 계층 프로토콜 : 

5계층 프로토콜(TCP/IP) 프로토콜 그룹에서 계층 구조: 하위 계층은 상위 계층의 일을 수행하기 위해 존재	
	
자세한것은 차후 배우게 될것이다.


	TPC/IP 프로토콜 그룹 응용계층
		- 계층간 논리적인 연결은 종단 대 종단
		- 메세지 교환
		- 프로세스 간 통신
		- HTTP,SMTP,email,telnet .....
		- DNS,IGMP

	TPC/IP 프로토콜 그룹 전송계층
		- 응용 계층에 서비스 제공 책임
		- TCP 
		- UDP

	TPC/IP 프로토콜 그룹 네트워크 계층
		-호스트-대-호스트
		-발신지에서 목적지까지 있는 경로상의 라우터들은 최선결로 채택
		- 인터넷 프로토콜
		-ICMP,iGMP,DHCP,ARP

	데이터 링크 계층
		- 데이터그램을 받아서 프레임이라는 패킷으로 캡슐화
		- 오류 탐지 및 교정 서비스 제공
	물리계층은 프레임을 받아서 0,1로 나눈다.

	캡슐화와 역캡슐화
		- 데이터 압축이라고 보면 되는건가?
